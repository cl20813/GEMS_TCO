# Pure-Space Fit Of Generated July ST Circulant Simulation Asset

This runbook is simulation-fitting only.  It does not generate simulated data.
First generate reusable simulated assets with:

```bash
/Users/joonwonlee/Documents/GEMS_TCO-1/Exercises/st_model/simulate_data/slurm_generate_july_st_circulant_real_locations_2022_2025.md
```

Then use this runbook to fit the generated 2024 real-location simulation pickle
with the same pure-space nugget diagnostic used for real data.

By default this file fits only the 2024 simulated July asset:

```bash
YEAR=2024
EXPECTED_HOURS=248
DATA_KIND="real_locations"
```

The asset itself is generated by 3D FFT circulant embedding in the separate
simulation generator.  The generator uses daily 8-hour 3D blocks, then samples
the generated field at each year's real July GEMS locations.  This fit script
does not run the FFT generator again.

```bash
SCRIPT="/home/jl2815/tco/exercise_25/st_model/day/pure_space/simulation/fit_sim_july2024_spatial_nugget_tiles.py"
DATA_PATH="/home/jl2815/tco/exercise_output/sim_data/july_st_circulant_realpattern/${YEAR}_july_st_circulant/sim_july${YEAR}_st_circulant_${DATA_KIND}.pkl"

    --time-col hour \
    --x-col Source_Longitude \
    --y-col Source_Latitude \
    --value-col ColumnAmountO3 \
    --coords raw \
    --smooth 0.5 \
    --neighbors 8 \
    --max-points 0 \
    --min-tile-points 80 \
    --tiles 3 \
    --device cuda \
    --summary-every 8
```

### Fit A Different Simulated Year

The generator creates one asset folder per year. To use another simulated year,
change only `YEAR` and `EXPECTED_HOURS` in the sbatch block. Keep the output
directory year-labelled so summaries do not mix.

Current generated July hour counts are:

```text
2022: 240
2023: 248
2024: 248
2025: 247
```

Examples:

```bash
# 2022
YEAR=2022
EXPECTED_HOURS=240

# 2023
YEAR=2023
EXPECTED_HOURS=248

# 2025
YEAR=2025
EXPECTED_HOURS=247
```

The existing path template already follows `YEAR`:

```bash
DATA_PATH="/home/jl2815/tco/exercise_output/sim_data/july_st_circulant_realpattern/${YEAR}_july_st_circulant/sim_july${YEAR}_st_circulant_${DATA_KIND}.pkl"
OUTDIR="/home/jl2815/tco/exercise_output/eda/simulation/${YEAR}_july_st_circulant_${DATA_KIND}_nugget3x3"
```

So the summaries remain separated, for example:

```text

2023_july_st_circulant_real_locations_nugget3x3
2024_july_st_circulant_real_locations_nugget3x3
```

### Optional: Random Day Subset Across 2022-2025

This is worth doing as a quick robustness check, but not as the main
stationarity diagnostic. It is valid only if we treat simulated days across
2022-2025 as iid draws from the same truth; in this generator that is mostly
reasonable because the truth is fixed, but the real-location masks and missing
hour patterns still differ by year. For the main result, prefer fitting each
year's full July asset separately.

Use this sbatch when you want a smaller mixed-year sample, e.g. 30 random days
from all available simulated July days. It first builds per-year manifests,
keeps only the sampled days, fits each selected hour, then summarizes all fitted
hours into one mixed output folder.

```bash
cd ./jobscript/tco/gp_exercise
nano fit_sim_random_days_2022_2025_nugget3x3_051026.sh
sbatch fit_sim_random_days_2022_2025_nugget3x3_051026.sh
```

```bash
#!/bin/bash
#SBATCH --job-name=fit_sim_rand_sp
#SBATCH --output=/home/jl2815/tco/exercise_output/fit_sim_rand_sp_%j.out
#SBATCH --error=/home/jl2815/tco/exercise_output/fit_sim_rand_sp_%j.err
#SBATCH --time=4:00:00
#SBATCH --nodes=1
#SBATCH --ntasks=1
#SBATCH --cpus-per-task=4
#SBATCH --mem=32G
#SBATCH --partition=gpu-redhat
#SBATCH --gres=gpu:1

set -euo pipefail

module purge
module use /projects/community/modulefiles
module load anaconda/2024.06-ts840
module load cuda/12.1.0

eval "$(conda shell.bash hook)"
conda activate faiss_env

echo "Running on: $(hostname)"
nvidia-smi
echo "Current date and time: $(date)"

export PYTHONPATH="/home/jl2815/tco:${PYTHONPATH:-}"
export OMP_NUM_THREADS=1
export MKL_NUM_THREADS=1
export OPENBLAS_NUM_THREADS=1
export NUMEXPR_NUM_THREADS=1

SCRIPT="/home/jl2815/tco/exercise_25/st_model/day/pure_space/simulation/fit_sim_july2024_spatial_nugget_tiles.py"
BASE="/home/jl2815/tco/exercise_output/sim_data/july_st_circulant_realpattern"
DATA_KIND="real_locations"
N_RANDOM_DAYS=30
SEED=20240510
OUTDIR="/home/jl2815/tco/exercise_output/eda/simulation/random${N_RANDOM_DAYS}days_seed${SEED}_2022_2025_st_circulant_real_locations_nugget3x3"
export OUTDIR N_RANDOM_DAYS SEED

mkdir -p "${OUTDIR}"
export MPLCONFIGDIR="${OUTDIR}/.mplconfig"
mkdir -p "${MPLCONFIGDIR}"

# 1) Build full manifests for each generated year.
for YEAR in 2022 2023 2024 2025; do
    DATA_PATH="${BASE}/${YEAR}_july_st_circulant/sim_july${YEAR}_st_circulant_${DATA_KIND}.pkl"
    MANIFEST="${OUTDIR}/manifest_${YEAR}_full.csv"
    srun python "${SCRIPT}" --mode manifest \
        --input "${DATA_PATH}" \
        --output-dir "${OUTDIR}" \
        --manifest "${MANIFEST}" \
        --month "${YEAR}-07" \
        --expected-hours 0 \
        --time-col hour \
        --x-col Source_Longitude \
        --y-col Source_Latitude \
        --value-col ColumnAmountO3 \
        --coords raw
done

# 2) Sample random days across all years and write one filtered manifest per year.
python - <<'PY'
from pathlib import Path
import os
import pandas as pd

out = Path(os.environ['OUTDIR'])
n_days = int(os.environ['N_RANDOM_DAYS'])
seed = int(os.environ['SEED'])
frames = []
for year in [2022, 2023, 2024, 2025]:
    p = out / f'manifest_{year}_full.csv'
    d = pd.read_csv(p)
    d['year'] = year
    d['sample_day'] = d['year'].astype(str) + '-' + d['day_index'].astype(int).astype(str).str.zfill(2)
    frames.append(d)
all_manifest = pd.concat(frames, ignore_index=True)
days = pd.Series(sorted(all_manifest['sample_day'].unique()))
selected = days.sample(n=min(n_days, len(days)), random_state=seed).sort_values().tolist()
selected_df = pd.DataFrame({'sample_day': selected})
selected_df.to_csv(out / 'random_selected_days.csv', index=False)
print('Selected random days:')
print(selected_df.to_csv(index=False).strip())
for year, d in all_manifest[all_manifest['sample_day'].isin(selected)].groupby('year'):
    keep = d.drop(columns=['year', 'sample_day']).reset_index(drop=True)
    keep.to_csv(out / f'manifest_{year}_random_days.csv', index=False)
    print(f'year={year}, days={keep["day_index"].nunique()}, hours={len(keep)}')
PY

# 3) Fit selected hours year by year into the same OUTDIR.
for YEAR in 2022 2023 2024 2025; do
    DATA_PATH="${BASE}/${YEAR}_july_st_circulant/sim_july${YEAR}_st_circulant_${DATA_KIND}.pkl"
    MANIFEST="${OUTDIR}/manifest_${YEAR}_random_days.csv"
    if [[ ! -s "${MANIFEST}" ]]; then
        echo "No sampled days for YEAR=${YEAR}; skipping."
        continue
    fi
    N_HOURS=$(($(wc -l < "${MANIFEST}") - 1))
    for IDX in $(seq 0 $((N_HOURS - 1))); do
        srun python "${SCRIPT}" --mode fit \
            --input "${DATA_PATH}" \
            --output-dir "${OUTDIR}" \
            --manifest "${MANIFEST}" \
            --array-index "${IDX}" \
            --month "${YEAR}-07" \
            --time-col hour \
            --x-col Source_Longitude \
            --y-col Source_Latitude \
            --value-col ColumnAmountO3 \
            --coords raw \
            --smooth 0.5 \
            --neighbors 8 \
            --max-points 0 \
            --min-tile-points 80 \
            --tiles 3 \
            --device cuda
    done
done

# 4) Summarize all fitted selected hours together.
srun python "${SCRIPT}" --mode summarize \
    --output-dir "${OUTDIR}" \
    --tiles 3

echo "Current date and time: $(date)"
```

### Optional: Fit Gridded Simulation Instead

For the gridded generated pickle, change:

```bash
DATA_KIND="gridded"
--x-col Longitude
--y-col Latitude
```

The gridded pickle was created using the documented half-cell threshold:

```text
abs(source_lat - grid_lat) <= 0.044 / 2
abs(source_lon - grid_lon) <= 0.063 / 2
```
